
## 1. 2D → 3D 개념  
**2D + 원근법 = 3D 효과**  

실제 3D 이미지는 양쪽 눈의 정보를 결합하여 인식됨 (입체 시각)

### 원근법 (Perspective)
- 멀리 있는 물체는 작게, 가까운 물체는 크게 보이게 표현
- 색상변화, 텍스쳐, 조명, 쉐이딩 등 다양한 효과를 추가해서 보다 사실적인 3D 물체표현
- **소실점 (Vanishing Point, VP)** 존재
- 2D + 원근법 = 3D


### 축소법
- 강조 기법 중 하나
- 거리감 표현에 사용

## 2. 렌더링 (Rendering)  
3D 물체에 대한 도형적 정보를 2D 화면에 표현하는 과정
```
원근법에서 팝핍현상 제거를 위해 보이지 않는 선들을 제거함(은면 제거)

- 보이지 않는 면 제거 → **은면 제거 (Hidden Surface Removal)**
- 팝핑 현상 제거에 도움
팝핑현상이란??????????
이부분을 정리해야합니다 팝핑현상이뭔지 그리고 렌더링이랑 뭔 관계인지 그리고 원근ㄱ법이랑은 또 뭔 관계인지..
```

## 3. 색상 (Color)
- 완벽한 3D 효과를 위해서는 각 면에 색상 추가가 필요
- but 모든 면에 동일한 색을 사용하면 3D 효과 감소
- 각 면에 다른 색을 적용 → 입체감 증가

---

## 4. 조명과 그림자 (Lighting & Shadow)
- 조명의 효과로 각면의 색상 진하기 조절
- 조명의 위치와 물체의 위치, 각도 고려해 적절한 그림자 삽입

## 5. 텍스처 (Texture)
- 3D 그래픽에서 객체 표면에 입히는 2D 이미지
- 나무 결, 천의 재질, 벽 등과 같은 다양한 이미지를 3D 물체의 각 면에 적절하게 입혀줌

## 7. 쉐이딩 & 쉐이더

### 쉐이딩 (Shading) - 이론, 과정
광원, 카메라, 물체 등의 재질들을 고려해 물체의 색상, 그림자, 반사, 블렝딩, 투명도 등 효과를 조정하여 현실적인 그래픽을 표현하는 과정

### 쉐이더 (Shader) - 도구, 코드
- 쉐이딩 계산을 수행하는 CPU 혹은 GPU에서 실행되는 프로그램

👉 **쉐이더가 쉐이딩을 수행한다**

## 8. 그래픽 파이프라인 기본 개념

### Vertex / Point / Pixel
```
3d세계 -> 변환과정 -> 2d화면
vertax,point                 pixel
이 파트 이쁘게 보이게 하기~

또
2d에서의 벌택스와 포인트

3d에서의 벌택스와 포인트
이거는 노트에 이렇게만 적혀잇엇는데 뭘까
```
**Vertex**
- 3D 공간에서 도형의 꼭짓점에 대한 정보가 담겨있는 데이터
- point의 특수한 형태
- vertex 하나가 가진 정보
  - 위치(x,y,z) <-3d공간좌표
  - 색상
  - 법선벡터(빛 계산용)
  - 텍스처 좌표 etc

**Point**
- 단순하게 좌표 하나를 가리키는 개념
- Vertex보다 단순

**Pixel**
- 최종적으로 명시적인 화면 공간(디스플레이)에서 나타나는 색상 및 좌표 정보

## 9. Primitive (프리미티브)
화면을 그릴때 사용하는 기본적인 도형 단위  
- 여러 vertex들 연결해서 구성됨
- Point → Vertex 1개로 이루어진 primitive
- Line(선) → Vertex 2개를 연결해 만든 primitive
- Triangle(삼각형) → Vertex 3개를 연결해 만든 가장 기본적인 면 단위

---

## 10. 알리아싱 (Aliasing)
픽셀은 정사각형이므로 곡선이나 대각선이 계단처럼 보이는 현상이 발생함
- 해상도가 낮을수록 두드러짐

### 해결 방법
- **안티 알리아싱 (Anti-Aliasing)**
  → 알리아싱을 줄여서 부드러운 이미지 생성하는 기법

## 11. 2D 직교 좌표계 & 클리핑

### 클리핑 (Clipping)
직교 좌표계의 커다란 영역 중 일부를 잘라(클리핑) PC의 화면 중 창(윈도우)에 할당하는 것
- 일반적으로 2개의 클리핑 영역 이용
```
### 뷰포트 (Viewport)
- 실제 화면에 표시되는 영역
클리핑영역은 150x100
윈도우영역은 300x200
뷰포트는 창의 전체 영역으로 설정 .(따라서 뷰포트도 300x200)
클리핑영역은 150x100
윈도우영역은 300x200
뷰포트는 150x100 -> 창 영역 자체를차지하지 못하고 좌측 하단만 채움. (ex PIP)
뷰포트는 윈도우 내에 펴시될 이미지릐 크기를확대축소하는데 사용될수잇음

#### 예시
- 클리핑 영역: 150 × 100
- 윈도우: 300 × 200

- 뷰포트 = 300 × 200 → 전체 화면 사용
- 뷰포트 = 150 × 100 → 일부 영역만 표시 (PIP 느낌)

👉 뷰포트는 이미지 확대/축소 역할 가능
```

## 12. 3D 직교 좌표계 & 투영

### 직교 투영 (Orthographic Projection)
영역 내에 있는 모든 물체들이 거리 관계 없이 모두 같은 크기, 비율로 표현
- 그 영역을 벗어나는 것들은 그려지지 않음 (설계도같은느낌)

### 📌 원근 투영 (Perspective Projection)
물체가 얼마나 가까이 또는 멀리 있는가에 따라 2D 화면에 나타나는 크기가 달라짐.
- 가까이 있는 물체는 크게, 멀리 있는 물체는 작게 나타남
